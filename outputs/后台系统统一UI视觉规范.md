# 后台系统统一 UI 视觉规范

> 本文件保留 2026-06-29 用户提供的全站 UI 优化提示词，作为后续页面维护和新增页面的长期视觉基线。

请对整个后台系统进行统一的 UI 视觉优化。要求：只调整视觉样式（颜色、字体、字号、间距、圆角、阴影、边框），不改动任何页面的功能、文案和数据结构。目标是让整体显得专业、简洁、克制、统一。请严格遵循以下设计系统：

## 一、配色（最多 5 种颜色，全站统一，禁止一个页面绿、一个页面紫）

- 主色（品牌色 / 主按钮 / 选中态 / 链接 / Tab 下划线）：墨绿 `#0F5132` 系（以现有侧边栏绿色为基准，全站统一，不要再混入紫色）
- 中性色：背景 `#F7F8FA`，卡片白 `#FFFFFF`，主文字 `#1A1D1F`，次要文字 `#6B7280`，边框/分割线 `#E5E7EB`
- 状态色（仅用于语义，不用于装饰）：成功绿、警告橙、错误红，各 1 种即可
- 禁止：紫色、渐变色作为主色；禁止彩虹色标签

## 二、标签 / Tag 配色（重点，解决“太花”问题）

- 默认所有标签统一为：浅灰底 + 深灰字（或浅主色底 + 主色字），同一组标签同色系
- 不要给每个标签随机配不同颜色；只有需要表达“状态/语义”时才用状态色
- 标签统一：字号 `13px`、内边距 `2px 8px`、圆角 `6px`（用小圆角，不要大胶囊形）

## 三、字体与字号（建立层级）

- 字体：全站统一一套无衬线字体（如 Inter / 思源黑体），最多两种字重
- 页面大标题 `24px / 600`；卡片标题 `16px / 600`；正文 `14px / 400`；辅助说明 `13px / 400` 用次要文字色；统计数字 `28px / 700`
- 行高：正文 `1.5`；标题 `1.3`

## 四、间距（用 4 的倍数，统一节奏）

- 卡片内边距 `24px`；卡片之间间距 `16px`；区块之间 `24px`
- 表格行高统一，单元格上下内边距 `12px`；图标与文字间距 `8px`
- 同一行内元素垂直居中对齐，禁止参差不齐

## 五、圆角 / 边框 / 阴影（克制）

- 圆角统一：卡片 `12px`、按钮/输入框 `8px`、标签 `6px`
- 边框：`1px #E5E7EB`；卡片用极轻阴影（`0 1px 3px rgba(0,0,0,0.06)`），不要重阴影
- 按钮：主按钮=主色实底白字；次按钮=白底+灰边+深灰字；二者风格全站一致

## 六、组件一致性

- 所有页面的页头（面包屑+标题+描述）、卡片、表格、按钮、Tab、标签样式完全一致
- 表格：表头浅灰底、深灰字；行 hover 浅色高亮；操作按钮风格统一
- 选中态 / hover 态 / 禁用态三种交互状态全站统一

请基于以上规范，逐页统一现有界面，确保任意两个页面放在一起看不出风格差异。
