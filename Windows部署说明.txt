AI招聘管理平台 Windows 部署说明

一、环境准备
1. 安装 Python 3.11 或更高版本，并确保 `python` 命令可用。
2. 安装 PostgreSQL，并创建数据库 `hr_platform`。
3. 确认 Playwright 已安装完成。
4. 先把项目根目录下的 `.env.example` 复制为 `.env`，再配置好 OpenRouter 和数据库连接串。

二、.env 配置
项目根目录下的 `.env` 文件应包含以下内容：

OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_API_KEY=你的OpenRouterKey
OPENROUTER_MODEL=deepseek/deepseek-v4-flash
DATABASE_URL=postgresql://user_delivery:delivery_pass@localhost:5432/hr_platform

说明：
- 数据库连接串统一写在 `.env` 里，不要再改启动脚本。
- 如果数据库账号、密码、主机或端口不同，只改这一行。

三、安装依赖
在项目根目录打开命令行，执行：

python -m pip install -r requirements.txt

如果你使用虚拟环境，请先激活虚拟环境，再执行上面的命令。

四、依次执行脚本
项目根目录下已经准备好 Windows 脚本，建议按顺序执行：

1. 安装依赖：

windows\01-install-deps.bat

2. 初始化数据库表结构（只建表，不导数据）：

windows\02-init-db.bat

3. 启动后端服务：

windows\03-start-backend.bat

五、初始化数据库表结构
如果你不想走“依次执行脚本”，也可以单独执行：

init-db-windows.bat

或者在 PowerShell 中执行：

.\init-db-windows.ps1

该步骤会：
- 创建 public schema 下的业务表
- 创建 recruit schema 下的表结构
- 自动补齐缺失字段
- 不写入任何种子数据

六、启动后端服务
执行：

run-windows.bat

或者在 PowerShell 中执行：

.\run-windows.ps1

启动后默认访问地址：

http://<Windows机器IP>:8000

七、验证
浏览器访问：

http://127.0.0.1:8000/api/health

如果成功，会返回类似：

{"status":"ok","service":"hr-platform"}

八、常见问题
1. 如果提示找不到 python，请检查 Python 是否已安装并加入 PATH。
2. 如果数据库连接失败，请确认 PostgreSQL 已启动，且 `DATABASE_URL` 正确。
3. 如果表结构初始化失败，先确认数据库 `hr_platform` 已创建，账号 `user_delivery` 有权限。
4. 如果前端页面打不开，确认后端服务已经启动在 8000 端口，并且 Windows 防火墙已放行。

九、推荐执行顺序
1. 安装依赖
2. 初始化数据库表结构
3. 启动后端服务

命令顺序：

windows\01-install-deps.bat
windows\02-init-db.bat
windows\03-start-backend.bat
