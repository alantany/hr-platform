AI Recruitment Platform Windows Deployment Guide

1. Prerequisites
1) Install Python 3.11 or later and make sure `python` is available in PATH.
2) Install PostgreSQL 17 and create the database `hr_platform`.
3) Make sure Playwright is already installed.
4) Copy `.env.example` to `.env` in the project root and fill in your OpenRouter and database settings.

2. .env configuration
The project root `.env` file should contain:

OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_API_KEY=your_openrouter_key
OPENROUTER_MODEL=deepseek/deepseek-v4-flash
DATABASE_URL=postgresql://user_delivery:delivery_pass@localhost:5432/hr_platform

Notes:
- Keep the database connection string in `.env`.
- If your database host, port, username, or password is different, change only that line.

3. Install dependencies
Open Command Prompt in the project root and run:

python -m pip install -r requirements.txt

If you use a virtual environment, activate it first.

4. Run the Windows scripts in order
The project already includes Windows scripts. Run them in this order:

1) Install dependencies:
windows\01-install-deps.bat

2) Initialize database tables only:
windows\02-init-db.bat

3) Start the backend service:
windows\03-start-backend.bat

5. Initialize database tables manually
If you do not want to use the step-by-step scripts, you can run:

init-db-windows.bat

Or in PowerShell:

.\init-db-windows.ps1

This step will:
- Create business tables in the public schema
- Create tables in the recruit schema
- Add missing columns automatically
- Not insert any seed data

6. Start the backend service
Run:

run-windows.bat

Or in PowerShell:

.\run-windows.ps1

Default API address:

http://<Windows-machine-IP>:8000

7. Verify
Open:

http://127.0.0.1:8000/api/health

If successful, it returns:

{"status":"ok","service":"hr-platform"}

8. Common issues
1) If Python is not found, check that Python is installed and added to PATH.
2) If the database connection fails, make sure PostgreSQL is running and `DATABASE_URL` is correct.
3) If table initialization fails, make sure database `hr_platform` exists and `user_delivery` has permission.
4) If the frontend cannot load, confirm the backend is running on port 8000 and Windows Firewall allows it.

9. Recommended order
1) Install dependencies
2) Initialize database tables
3) Start the backend service

Command order:

windows\01-install-deps.bat
windows\02-init-db.bat
windows\03-start-backend.bat
