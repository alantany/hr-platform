# 模块八：评价体系

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：提供评价等级、简历评价、评价追溯和评价统计，沉淀候选人质量判断。

**本模块负责**：

- 评价等级管理
- 简历评价操作
- 评价记录追溯
- 评价统计

**本模块不负责**：

- 候选人主数据导入
- 岗位创建
- 权限角色维护

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

评价体系模块为招聘团队提供标准化的简历评价功能，支持评价等级管理、评价操作、评价记录追溯、评价统计和评价对比。

## 独立边界

### 上游依赖

- 求职者数据池
- 候选人跟踪与管理
- 标签字典管理
- 统计与分析
- 权限管理体系

### 本模块核心数据对象

- 评价等级
- 评价记录
- 评价人
- 候选人
- 评价统计

### 对外输出

- 面向其他模块提供经过权限过滤的数据查询结果。
- 对关键业务动作输出可追溯的状态变化和操作日志。
- 对通知、统计、AI 或导出模块输出明确的业务事件或结果数据。

## 功能拆解

### 1. 评价等级管理

- 功能描述：完成业务数据的操作管理。进入评价体系 → 选择「评价等级管理」→ 查看/新增/编辑/删除评价等级 → 保存
- 菜单/入口：评价体系 → 评价等级管理
- 界面元素：等级名称、分值、说明、颜色标识、排序、状态（启用/禁用）
- 输出结果：更新后的评价等级体系
- 权限说明：仅超级管理员可管理评价等级

**操作流程**

1. 进入评价体系
2. 选择「评价等级管理」
3. 查看/新增/编辑/删除评价等级
4. 保存

**输入字段**

| 字段 | 类型 | 必填 | 校验规则 |
| --- | --- | --- | --- |
| 等级名称 | 文本 | 必填 |  |
| 分数范围 | 枚举 | 必填 | 枚举值-优秀（5分，绿色）、良好（4分，蓝色）、一般（3分，橙色）、较差（2分，黄色）、不合格（1分，粉色） |
| 等级颜色 | 枚举 | 必填 | 枚举值-优秀（5分，绿色）、良好（4分，蓝色）、一般（3分，橙色）、较差（2分，黄色）、不合格（1分，粉色） |

**业务规则**

- 预置等级：优秀（90分，绿色）、良好（75分，黄色）、一般（60分，橙色）、较差（40分，红色）；支持新增/编辑/删除等级；删除前检查是否有关联评价记录；编辑后实时同步到所有关联模块

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。
- 关键操作写入操作日志；涉及数据列表时必须按角色数据范围过滤。
- 与其他模块联动时，状态变化和输出数据可被后续模块消费。

## 推荐接口与后端边界

- 列表查询接口必须支持分页、排序、筛选、关键词搜索和数据权限过滤。
- 详情接口必须校验当前用户是否可访问该业务对象。
- 新增/编辑/删除/状态流转接口必须记录操作日志。
- 批量、导入、导出、AI、通知等长任务建议建任务表或事件表，保留状态、错误信息和重试线索。
- 所有枚举值优先从产品规格和标签字典中抽取，不允许前后端各自硬编码不同口径。

## 数据与权限要求

- 字段定义以 `docs/prd/90-data-fields.md` 为准；模块内表格字段是功能输入/输出的补充。
- 功能权限、数据权限、菜单可见性以 `docs/prd/91-permission-matrix.md` 为准。
- 菜单隐藏不能替代后端权限校验；所有 API 都必须执行权限判断。
- 导出、统计、AI 推荐结果、通知跳转必须与列表/详情使用同一套数据权限过滤。

## 测试建议

- 功能级：逐个功能点覆盖主流程、必填校验、失败提示和操作日志。
- 权限级：超级管理员、组长、操作员分别验证可见菜单、可访问数据、可执行动作。
- 数据级：验证字段类型、枚举值、关联关系、唯一性和脱敏/加密要求。
- 集成级：验证本模块与上游/下游模块的事件、状态和数据联动。

##### 1、业务需求

评价等级管理是评价体系模块为招聘团队提供标准化的简历评价功能，支持评价等级管理、评价操作、评价记录追溯、评价统计和评价对比。中的重要功能组件。该功能主要解决以下业务问题：

在人力公司招聘管理业务场景中，评价体系需要此功能来标准化简历评价流程。通过评价等级管理功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

评价等级管理功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如标签字典管理、评价体系、候选人跟踪表）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

评价等级管理功能定位于招聘管理的核心功能，是评价等级管理体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持业务数据的完整管理
2. 支持业务数据的查询和操作
3. 操作结果实时生效

评价等级管理功能与其他功能的关联关系：该功能与与其他业务模块联动形成完整业务流程紧密关联，共同支撑招聘管理的核心业务流程的完整业务流程。通过评价等级管理功能，操作员可以完成业务数据的操作管理，为后续的后续的推荐、交付和管理工作奠定数据基础。

界面元素： 评价等级（优秀（5分，绿色）、良好（4分，蓝色）、一般（3分，橙色）、较差（2分，黄色）、不合格（1分，粉色））、关联面试轮次、备注

操作流程：
1. 在简历详情页
2. 点击「新增评价」
3. 选择评价等级
4. 关联面试轮次
5. 填写备注
6. 保存


输入字段：

<!-- table:44 rows:4 cols:4 -->
| 字段 | 类型 | 必填 | 校验规则 |
| --- | --- | --- | --- |
| 关联面试轮次 | 枚举 | 必填 | 通过候选人跟踪表的面试轮次-面试结果判断，让最后一条面试信息的面试结果为通过后，自动向关联面试轮次赋值：第XX轮 |
| 评价等级 | 枚举 | 必填 | 枚举值-优秀（5分，绿色）、良好（4分，蓝色）、一般（3分，橙色）、较差（2分，黄色）、不合格（1分，粉色） |
| 评价内容 | 文字 | 必填 |  |

输出结果： 新增的评价记录

业务规则：
- 评价等级从标签字典选择；评价关联具体面试轮次（可指定第几轮面试）；备注可填写多条；支持对同一简历多次评价

权限说明： 所有用户按数据权限范围操作

#### 评价记录追溯

##### 1、业务需求

评价记录追溯是评价体系模块为招聘团队提供标准化的简历评价功能，支持评价等级管理、评价操作、评价记录追溯、评价统计和评价对比。中的重要功能组件。该功能主要解决以下业务问题：

在人力公司招聘管理业务场景中，评价体系需要此功能来标准化简历评价流程。通过评价记录追溯功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

评价记录追溯功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如标签字典管理、评价体系、候选人跟踪表）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

评价记录追溯功能定位于招聘管理的核心功能，是评价记录追溯体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持业务数据的完整管理
2. 支持业务数据的查询和操作
3. 操作结果实时生效

评价记录追溯功能与其他功能的关联关系：该功能与与其他业务模块联动形成完整业务流程紧密关联，共同支撑招聘管理的核心业务流程的完整业务流程。通过评价记录追溯功能，操作员可以完成业务数据的操作管理，为后续的后续的推荐、交付和管理工作奠定数据基础。

功能权限方面，该功能有明确的角色权限限制。所有操作均记录操作日志，确保操作的可追溯性。

评价体系 → 评价记录追溯

### 1. 评价记录追溯操作

### 1. 评价统计操作
功能描述： 完成业务数据的操作管理。在简历详情 → 查看评价统计 → 显示平均评分、最高/最低评分、评价次数、最近评价

界面元素： 简历ID

操作流程：
1. 在简历详情
2. 查看评价统计
3. 显示平均评分、最高/最低评分、评价次数、最近评价


输入字段：
枚举值-优秀（5分，绿色）、良好（4分，蓝色）、一般（3分，橙色）、较差（2分，黄色）、不合格（1分，粉色）
输出结果： 评价统计结果

业务规则：
- 平均评分：该简历所有评价的平均分值；最高/最低评分：该简历所有评价的最高/最低分值；评价次数：该简历的评价记录总数；最近评价：最近一次评价的时间和等级
