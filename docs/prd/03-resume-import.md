# 模块三：简历导入

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：支持简历单文件和批量导入，并将简历解析为可检索、可管理的结构化数据。

**本模块负责**：

- 单文件导入
- 批量导入
- 结构化存储
- 导入结果和失败反馈

**本模块不负责**：

- 候选人后续推荐交付
- 复杂 AI 匹配策略
- 客户/项目/岗位管理

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

简历导入是招聘管理系统的求职者简历获取入口，可通过手动导入/批量导入方式将简历导入系统中，并保存导入的操作记录。

## 独立边界

### 上游依赖

- 求职者数据池
- AI能力中心
- 权限管理体系
- 通知提醒

### 本模块核心数据对象

- 简历文件
- 导入任务
- 结构化简历字段
- 导入日志
- 解析结果

### 对外输出

- 面向其他模块提供经过权限过滤的数据查询结果。
- 对关键业务动作输出可追溯的状态变化和操作日志。
- 对通知、统计、AI 或导出模块输出明确的业务事件或结果数据。

|---:|---|---|---|---|

| 2 | 简历批量导入 | 求职者数据池 → 简历导入 → 批量导入 | 批量结构化存储的简历数据 | 有简历导入权限的用户均可操作 |

## 功能拆解

### 1. 简历单文件导入

- 功能描述：高效管理和运营候选人资源。进入简历导入页面 → 选择单文件上传（Word/PDF）→ 系统自动解析 → 提取结构化信息 → 执行去重对比（姓名+身份证号 或 姓名+手机号）→ 若存在相同简历弹出对比页面 → 操作者判断是否覆盖 → 导入成功
- 菜单/入口：求职者数据池 → 简历导入 → 单文件导入
- 界面元素：简历文件（.doc/.docx/.pdf）
- 输出结果：结构化存储的简历数据、原始文件附件
- 权限说明：有简历导入权限的用户均可操作

**操作流程**

1. 进入简历导入页面
2. 选择单文件上传（Word/PDF）
3. 系统自动解析
4. 提取结构化信息
5. 执行去重对比（姓名+身份证号 或 姓名+手机号）
6. 若存在相同简历弹出对比页面
7. 操作者判断是否覆盖
8. 导入成功

**输入字段**

<!-- table:16 rows:2 cols:4 -->
| 字段 | 类型 | 必填 | 校验规则 |
| --- | --- | --- | --- |
| 文件导入 | 结构化文件 | - | Doc、docX、pdf格式的校验，为此格式报错：“格式文件不符，请重新上传” |

**业务规则**

- 支持Word和PDF格式；自动提取姓名、身份证号、手机号、邮箱、年龄、性别、学历等18+结构化字段；去重依据：姓名+身份证号 或 姓名+手机号 或 姓名+手机号+教育经历；相同简历弹出对比页面，操作者可判断是否覆盖；导入后简历状态为「激活」；导入文件带水印

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。
- 关键操作写入操作日志；涉及数据列表时必须按角色数据范围过滤。
- 与其他模块联动时，状态变化和输出数据可被后续模块消费。

### 2. 简历批量导入

- 功能描述：高效管理和运营候选人资源。进入简历导入页面 → 批量上传简历文件 → 系统解析所有文件 → 执行去重对比 → 显示统计结果（预期导入数/入库数/重复数）→ 点击数字查看对比列表 → 操作者确认后批量入库
- 菜单/入口：求职者数据池 → 简历导入 → 批量导入
- 界面元素：多个简历文件（.doc/.docx/.pdf）
- 输出结果：批量结构化存储的简历数据
- 权限说明：有简历导入权限的用户均可操作

**操作流程**

1. 进入简历导入页面
2. 批量上传简历文件
3. 系统解析所有文件
4. 执行去重对比
5. 显示统计结果（预期导入数/入库数/重复数）
6. 点击数字查看对比列表
7. 操作者确认后批量入库

**输入字段**

<!-- table:17 rows:2 cols:4 -->
| 字段 | 类型 | 必填 | 校验规则 |
| --- | --- | --- | --- |
| 文件导入 | ZIP压缩包 | - | 需要解析zip压缩包并对压缩包内的Doc、docX、pdf格式的校验，为此格式报错：“格式文件不符，请重新上传” |

**业务规则**

- 预期导入数 = 成功解析的文件数；入库数 = 数据库无此简历数；重复数 = 数据库存在相同简历数；点击数字可弹出具体对比列表；单击列表项弹出简历对比信息；导入后状态为「激活」

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。
- 关键操作写入操作日志；涉及数据列表时必须按角色数据范围过滤。
- 与其他模块联动时，状态变化和输出数据可被后续模块消费。

### 3. 简历结构化存储

- 功能描述：高效管理和运营候选人资源。系统自动解析简历文件 → 提取结构化字段 → 写入数据库 → 建立索引
- 菜单/入口：求职者数据池 → 简历管理 → 结构化存储
- 界面元素：简历原始文件
- 输出结果：结构化数据库记录
- 权限说明：系统自动执行

**操作流程**

1. 系统自动解析简历文件
2. 提取结构化字段
3. 写入数据库
4. 建立索引

## 推荐接口与后端边界

- 列表查询接口必须支持分页、排序、筛选、关键词搜索和数据权限过滤。
- 详情接口必须校验当前用户是否可访问该业务对象。
- 新增/编辑/删除/状态流转接口必须记录操作日志。
- 批量、导入、导出、AI、通知等长任务建议建任务表或事件表，保留状态、错误信息和重试线索。
- 所有枚举值优先从产品规格和标签字典中抽取，不允许前后端各自硬编码不同口径。

## 数据与权限要求

- 字段定义以 `docs/prd/90-data-fields.md` 为准；模块内表格字段是功能输入/输出的补充。
- 功能权限、数据权限、菜单可见性以 `docs/prd/91-permission-matrix.md` 为准。
- 菜单隐藏不能替代后端权限校验；所有 API 都必须执行权限判断。
- 导出、统计、AI 推荐结果、通知跳转必须与列表/详情使用同一套数据权限过滤。

- 集成级：验证本模块与上游/下游模块的事件、状态和数据联动。

简历导入是招聘管理系统的求职者简历获取入口，可通过手动导入/批量导入方式将简历导入系统中，并保存导入的操作记录。

### 功能规格详情

#### 简历单文件导入

##### 1、业务需求

简历单文件导入是求职者数据池是招聘管理系统的核心模块，负责统一管理所有候选人的简历数据。该模块提供简历的存储、检索、筛选、跟踪、评价和状态管理的完整功能，支持AI简历解析和自动匹配能力。中的重要功能组件。该功能主要解决以下业务问题：

在人力公司招聘管理业务场景中，求职者数据管理需要此功能来高效运营候选人资源。通过简历单文件导入功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

简历单文件导入功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如标签字典管理、项目管理、候选人跟踪表）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

简历单文件导入功能定位于求职者数据管理的核心功能，是简历单文件导入体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持简历的高效导入、结构化存储和快速检索
2. 支持多维度条件筛选、状态管理和评价功能
3. 操作结果实时生效，数据自动更新

简历单文件导入功能与其他功能的关联关系：该功能与与标签字典、项目管理、候选人跟踪表联动提供完整的候选人管理紧密关联，共同支撑候选人数据的全生命周期管理流程的完整业务流程。通过简历单文件导入功能，操作员可以高效管理和运营候选人资源，为后续的候选人推荐、面试和交付工作奠定数据基础。

功能权限方面，该功能支持不同角色的差异化数据访问，管理员全量、组长授权范围、操作员个人范围。所有操作均记录操作日志，确保操作的可追溯性。

在人力公司招聘管理业务场景中，求职者数据管理需要此功能来高效运营候选人资源。通过简历结构化存储功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

简历结构化存储功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如标签字典管理、项目管理、候选人跟踪表）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

简历结构化存储功能定位于求职者数据管理的核心功能，是简历结构化存储体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持简历的高效导入、结构化存储和快速检索
2. 支持多维度条件筛选、状态管理和评价功能
3. 操作结果实时生效，数据自动更新

简历结构化存储功能与其他功能的关联关系：该功能与与标签字典、项目管理、候选人跟踪表联动提供完整的候选人管理紧密关联，共同支撑候选人数据的全生命周期管理流程的完整业务流程。通过简历结构化存储功能，操作员可以高效管理和运营候选人资源，为后续的候选人推荐、面试和交付工作奠定数据基础。
