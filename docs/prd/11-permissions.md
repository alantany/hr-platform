# 模块十一：权限管理体系

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：管理用户、角色、功能权限、数据权限、候选人归属保护和操作日志，作为全系统安全底座。

**本模块负责**：

- 用户创建/编辑/启禁用/重置密码
- 角色创建/编辑/删除
- 功能权限配置
- 数据权限分配
- 候选人归属保护
- 操作日志管理

**本模块不负责**：

- 具体业务模块的业务规则
- AI 算法
- 统计图表口径

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

采用RBAC（基于角色的访问控制）+ 数据级权限双层控制模型，为系统提供细粒度的权限管理能力。支持功能权限（菜单/按钮）和数据权限（公司/项目/岗位三级粒度）的灵活配置，确保系统数据的安全性和操作的合规性。新增候选人归属保护机制，防止候选人被恶意抢占。

## 功能拆解

### 1. 用户管理-创建用户

- 菜单/入口：系统设置 → 权限管理 → 用户管理
- 输出结果：新建用户账号并绑定角色、状态和基础信息。
- 权限说明：仅超级管理员可创建用户；创建后用户权限由绑定角色和数据权限共同决定。

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。
- 关键操作写入操作日志；涉及数据列表时必须按角色数据范围过滤。
- 与其他模块联动时，状态变化和输出数据可被后续模块消费。

### 2. 用户管理-编辑用户

- 功能描述：管理员修改已创建用户的基本信息，包括姓名、联系方式、角色、状态等。角色变更时系统自动重新计算数据权限。修改操作实时生效并记录操作日志。
- 菜单/入口：系统设置 → 权限管理 → 用户管理
- 界面元素：编辑表单（同创建表单，但用户名不可编辑）、保存按钮、取消按钮
- 输出结果：用户信息更新，权限自动重新计算
- 权限说明：仅超级管理员（ADMIN）可执行编辑用户操作。

**操作流程**

1. 在用户列表中点击某用户行的「编辑」按钮
2. 在弹窗中修改用户信息（用户名不可修改）
3. 可更换用户角色（自动重新计算权限）
4. 点击「保存」提交修改
5. 系统校验通过后保存变更

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。
- 关键操作写入操作日志；涉及数据列表时必须按角色数据范围过滤。
- 与其他模块联动时，状态变化和输出数据可被后续模块消费。

### 3. 用户管理-禁用/启用用户

- 功能描述：管理员可将用户账号禁用或启用。禁用后用户无法登录系统，启用后恢复登录能力。禁用操作需二次确认，确保操作安全。
- 菜单/入口：系统设置 → 权限管理 → 用户管理
- 界面元素：操作列中的禁用/启用按钮、确认弹窗
- 输出结果：用户账号状态变更
- 权限说明：仅超级管理员（ADMIN）可执行禁用/启用操作。

**操作流程**

1. 在用户列表中定位目标用户
2. 点击操作列的「禁用」或「启用」按钮
3. 弹出确认对话框，显示操作影响说明
4. 点击确认执行操作
5. 禁用后用户列表中该用户状态标签变为红色「禁用」

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。

### 4. 用户管理-重置密码

- 功能描述：管理员在用户忘记密码或安全策略要求时，为该用户重置登录密码。新密码满足复杂度要求并通过系统通知用户。
- 菜单/入口：系统设置 → 权限管理 → 用户管理
- 界面元素：密码重置弹窗（新密码输入框、确认密码输入框、发送通知选项）
- 输出结果：用户密码更新成功
- 权限说明：仅超级管理员（ADMIN）可执行密码重置操作。

**操作流程**

1. 在用户列表中定位目标用户
2. 点击操作列的「重置密码」按钮
3. 在弹窗中输入新密码并确认
4. 选择通知方式（站内消息/短信/不通知）
5. 点击「确认重置」提交
6. 密码修改成功后，用户下次登录可使用新密码

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。

### 5. 角色管理-创建角色

- 界面元素：角色名称（唯一）、角色描述、功能权限（菜单+按钮勾选）、数据权限范围、状态

- 角色名称唯一；预置角色：超级管理员（不可删除）、组长、操作员；功能权限通过树形结构勾选

### 12. 操作日志管理

- 功能描述：管理系统的用户和权限配置。进入操作日志 → 支持按时间/用户/操作类型筛选 → 查看日志 → 支持导出
- 菜单/入口：系统设置 → 权限管理 → 操作日志管理
- 界面元素：筛选条件：操作人、操作模块、操作类型、时间范围、IP地址
- 输出结果：操作日志列表
- 权限说明：仅超级管理员可查看操作日志

**操作流程**

1. 进入操作日志
2. 支持按时间/用户/操作类型筛选
3. 查看日志
