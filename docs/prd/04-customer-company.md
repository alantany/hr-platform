# 模块四：客户公司管理

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：维护客户公司、客户项目、客户岗位和岗位候选人列表，支撑招聘需求进入系统。

**本模块负责**：

- 客户公司列表、创建、维护
- 客户项目创建/维护
- 客户岗位创建/维护
- 紧急岗位、岗位候选人列表、岗位关闭

**本模块不负责**：

- 求职者数据池基础检索
- 用户角色配置
- AI 能力实现细节

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

客户公司管理模块用于管理招聘系统的客户基本信息，提供客户公司的增删改查、项目与岗位树状展开、招聘进度监控等功能。

## 独立边界

### 上游依赖

- 权限管理体系
- 项目管理
- 求职者数据池
- 标签字典管理
- 通知提醒

### 本模块核心数据对象

- 客户公司
- 联系人
- 项目
- 岗位
- 岗位候选人
- 紧急程度
- 岗位状态

### 对外输出

- 面向其他模块提供经过权限过滤的数据查询结果。
- 对关键业务动作输出可追溯的状态变化和操作日志。
- 对通知、统计、AI 或导出模块输出明确的业务事件或结果数据。

## 功能拆解

### 1. 客户公司列表

- 功能描述：完成业务数据的操作管理。进入客户公司管理 → 查看公司列表 → 支持按公司名称搜索 → 悬浮鼠标显示项目列表 → 点击项目展开岗位列表 → 点击岗位跳转候选人列表
- 菜单/入口：系统设置 → 客户公司列表
- 界面元素：无（系统自动加载）
- 输出结果：客户公司列表（含项目/岗位树状结构）
- 权限说明：管理员全量数据；组长授权范围；操作员仅分配的

**操作流程**

1. 进入客户公司管理
2. 查看公司列表
3. 支持按公司名称搜索
4. 悬浮鼠标显示项目列表
5. 点击项目展开岗位列表
6. 点击岗位跳转候选人列表

**业务规则**

- 列表字段：公司名称、状态（招聘中/空闲/失效）、联系人、联系电话、地址、项目数、岗位数、招聘进度；紧急岗位优先展示并显示倒计时；支持树状展开/折叠
