# 模块五：项目管理

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：统一管理项目与岗位需求，支持 JD 生成岗位、手动创建岗位、分配操作员和候选人筛选锁定。

**本模块负责**：

- 项目列表和岗位列表
- JD 单生成岗位
- 手动创建岗位
- 多选操作员权限分配
- 岗位候选人筛选与锁定

**本模块不负责**：

- 客户公司基础信息维护
- 推荐结果和交付记录
- 系统级权限菜单配置

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

项目管理模块作为客户管理的下级模块，用于统一管理所有客户项目与岗位需求，支持JD单解析生成岗位、多选操作员权限分配、岗位关闭开关等能力。

## 推荐接口与后端边界

- 列表查询接口必须支持分页、排序、筛选、关键词搜索和数据权限过滤。
- 详情接口必须校验当前用户是否可访问该业务对象。
- 新增/编辑/删除/状态流转接口必须记录操作日志。
- 批量、导入、导出、AI、通知等长任务建议建任务表或事件表，保留状态、错误信息和重试线索。
- 所有枚举值优先从产品规格和标签字典中抽取，不允许前后端各自硬编码不同口径。

#### 项目列表Tab

##### 1、业务需求

项目列表Tab是项目管理模块作为客户管理的下级模块，用于统一管理所有客户项目与岗位需求，支持JD单解析生成岗位、多选操作员权限分配、岗位关闭开关等能力。中的重要功能组件。该功能主要解决以下业务问题：

在人力公司招聘管理业务场景中，项目管理需要此功能来组织岗位需求和团队分配。通过项目列表Tab功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

项目列表Tab功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如客户公司管理、项目管理、权限管理）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

项目列表Tab功能定位于招聘管理的核心功能，是项目列表Tab体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持业务数据的完整管理
2. 支持业务数据的查询和操作

### 1. 岗位候选人筛选与锁定操作
功能描述： 完成业务数据的操作管理。进入岗位候选人列表 → 筛选简历 → 系统自动锁定被筛选的简历 → 标记为对应操作员的筛选简历 → 更新配额计数

界面元素： 筛选条件、简历ID

操作流程：
1. 进入岗位候选人列表
2. 筛选简历
3. 系统自动锁定被筛选的简历
4. 标记为对应操作员的筛选简历
5. 更新配额计数
