# 模块一：首页-数据看板

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：为超级管理员、组长、操作员提供差异化首页工作台，汇总关键指标、待办、快捷入口、排行和下钻入口。

**本模块负责**：

- 角色化看板布局与数据卡片
- 日期面板/周历看板和三级下钻
- 快捷入口、待办、排行、效率概览

**本模块不负责**：

- 业务数据的原始维护
- 权限配置本身
- 统计口径之外的临时图表

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

首页-数据看板模块是系统的核心入口模块，根据不同角色（管理员、组长、操作员）展示差异化的工作台页面。该模块通过统计卡片、日期面板、周历看板、快捷入口、待办事项、绩效排行等组件，为各角色提供个性化的数据汇总和快捷操作入口，帮助管理者快速了解运营状态，提高日常工作效率。

## 独立边界

### 上游依赖

- 权限管理体系
- 求职者数据池
- 项目管理
- 推荐与交付管理
- 统计与分析
- 通知提醒

### 本模块核心数据对象

- 用户角色
- 统计指标
- 待办事项
- 操作日志
- 通知
- 客户/项目/岗位/简历聚合数据

### 对外输出

- 面向其他模块提供经过权限过滤的数据查询结果。
- 对关键业务动作输出可追溯的状态变化和操作日志。
- 对通知、统计、AI 或导出模块输出明确的业务事件或结果数据。

## 推荐接口与后端边界

- 列表查询接口必须支持分页、排序、筛选、关键词搜索和数据权限过滤。
- 详情接口必须校验当前用户是否可访问该业务对象。
- 新增/编辑/删除/状态流转接口必须记录操作日志。
- 批量、导入、导出、AI、通知等长任务建议建任务表或事件表，保留状态、错误信息和重试线索。
- 所有枚举值优先从产品规格和标签字典中抽取，不允许前后端各自硬编码不同口径。

## 数据与权限要求

- 字段定义以 `docs/prd/90-data-fields.md` 为准；模块内表格字段是功能输入/输出的补充。
- 功能权限、数据权限、菜单可见性以 `docs/prd/91-permission-matrix.md` 为准。
- 菜单隐藏不能替代后端权限校验；所有 API 都必须执行权限判断。
- 导出、统计、AI 推荐结果、通知跳转必须与列表/详情使用同一套数据权限过滤。

- 数据级：验证字段类型、枚举值、关联关系、唯一性和脱敏/加密要求。
- 集成级：验证本模块与上游/下游模块的事件、状态和数据联动。

## 模块一：首页-数据看板

首页-数据看板模块是系统的核心入口模块，根据不同角色（管理员、组长、操作员）展示差异化的工作台页面。该模块通过统计卡片、日期面板、周历看板、快捷入口、待办事项、绩效排行等组件，为各角色提供个性化的数据汇总和快捷操作入口，帮助管理者快速了解运营状态，提高日常工作效率。

### 功能规格详情

#### 首页-数据看板（管理员）

##### 1、业务需求

作为管理员，需要在全局视角下掌握系统运营的整体状况。首页数据看板为管理员提供系统核心运营指标的实时概览，包括简历总量、今日新增、活跃简历、待处理任务、本月推荐、本月录用等关键指标的汇总展示。通过日期面板和周历看板，管理员可以逐日或逐周查看运营趋势，并支持下钻到明细数据，支持三级下钻链路（汇总->明细->条件筛选），帮助管理员快速发现异常、做出决策。管理员待办事项和系统效率概览则帮助管理员及时处理重要事项，确保系统正常运转。

##### 2、功能说明

首页-数据看板（管理员）是系统的核心入口页面，面向超级管理员角色。该功能模块通过三个核心区域为管理员提供系统运营全局视图：（1）顶部统计卡片区域展示简历总量、今日新增、活跃简历、待处理任务、本月推荐、本月录用等6项核心指标，并显示与昨日/上月的对比趋势；（2）中部日期面板/周历看板区域，支持按日/周切换，展示今日面试、今日推荐、今日入职、客户合同终止等汇总数据，点击汇总卡片可下钻查看明细并支持多级条件筛选；（3）底部区域包括快捷入口（12个常用功能模块入口）、系统效率概览、组长绩效排行、系统设置快捷入口、最近操作日志和管理员待办事项列表。每个汇总指标均可点击下钻，形成"汇总->明细表->条件筛选列"的三级数据穿透链路。页面根据管理员角色展示专属的全局运营指标，包括客户公司数、岗位数、求职者数量等系统级统计项。

###### 4.3 功能项详细描述

首页-数据看板（管理员）功能项包含以下11个功能点，覆盖管理员工作台的核心交互：

1. 统计卡片区域——展示6个核心指标（简历总量、今日新增、活跃简历、待处理任务、本月推荐、本月录用）的实时汇总数据，支持点击下钻到明细数据，卡片显示与昨日/上月的对比趋势。

2. 日期面板视图——按日查看运营指标汇总，支持左右箭头翻页和快速导航到当天或本周，面板显示今日面试、今日推荐、今日入职、客户合同终止等汇总数据，以及客户公司数、岗位数、求职者数量、总推荐数、总操作员数等通用统计项。

3. 周历看板视图——按周对比运营指标趋势，显示本周7天的每日推荐、面试、入职等数据汇总，以及周汇总卡片（本周客户公司数、本周岗位数、本周求职者数量、本周总推荐数、本周总操作员数），支持日期切换。

4. 快捷入口——12个常用功能模块的快捷访问按钮，包括求职者数据池、简历导入、客户管理、项目管理、标签字典、统计管理、通知中心、用户管理、角色管理、权限管理、质保期管理、评价管理，点击跳转对应功能模块。

5. 系统效率概览——展示系统整体运营效率指标，包括总操作员数、今日活跃操作员数、今日简历导入数、今日推荐数、平均响应时间、简历解析成功率6项指标。

6. 组长绩效排行——展示各组长团队的绩效排名，按团队推荐量、转化率等指标综合排序，展示Top 5组长团队数据，帮助管理员了解团队表现。

7. 系统设置入口——6个系统管理功能的快捷入口，包括系统配置、操作日志、邮件服务器、响应式适配、多租户配置、个人账号设置，方便管理员快速访问系统管理功能。

8. 最近操作日志——展示最近5条系统操作记录，包括操作人、操作时间、操作模块、操作内容和操作结果，帮助管理员了解系统最新动态。

9. 管理员待办事项——展示需要管理员处理的待办任务列表，包括面试提醒、简历推荐、质保到期、评价待处理、岗位匹配、合同到期、入职提醒等7种类型的待办事项，高优先级置顶显示。

10. 日期面板汇总下钻——点击日期面板中的任意汇总卡片，下钻查看对应明细数据，如点击"今日面试"下钻到面试明细表，点击"今日推荐"下钻到推荐明细表。

11. 三级下钻筛选——明细表->条件筛选列的多级数据穿透，点击明细表中的任意一行可下钻至条件筛选列，支持多条件组合筛选（AND关系），筛选结果以标签形式展示已选条件，支持清空和重置筛选。

#### 首页-数据看板（组长）

##### 1、业务需求

作为团队管理者，组长需要在日常工作中快速了解团队的整体运营状况和成员的工作表现。首页数据看板为组长展示其授权范围内的团队运营数据，包括简历总量、今日新增、团队推荐总量、团队转化率、客户公司数和项目数等6项核心指标。通过日期面板，组长可以查看今日组员获取简历数量、团队推荐简历数、客户处理岗位数等团队级指标。待办事项模块帮助组长及时处理待审核的简历推荐、紧急岗位处理等任务。下属操作员绩效排行帮助组长掌握团队成员的工作量和效率，负责客户公司列表帮助组长了解项目进度。

##### 2、功能说明

首页-数据看板（组长）是面向组长角色的工作台页面。该功能模块为组长提供其授权范围内团队运营数据的集中展示：（1）顶部统计卡片展示简历总量、今日新增、团队推荐总量、团队转化率、客户公司数、项目数等6项团队指标，并显示对比趋势；（2）日期面板区域支持按日/周切换，展示今日组员获取、今日推荐简历、今日客户处理岗位、今日客户数、今日项目数等团队指标，每项均可下钻到组员明细；（3）快捷入口（8个常用功能模块）；（4）下属操作员绩效排行展示组员的工作效率和推荐量排名；（5）负责客户公司列表展示团队负责的客户项目进度；（6）项目岗位进度以进度条形式展示各项目岗位的完成状态；（7）组长待办事项列表展示需要紧急审核的推荐、新匹配简历待审核、低于平均的组员需跟进等提醒。页面展示的数据范围严格受组长数据权限控制，仅显示授权范围内的团队数据。

该功能的界面布局参考原型页面dashboard.html：首页工作台页面采用多角色切换的工作台布局。页面顶部为统计卡片区域（6个卡片横向排列：简历总量、今日新增、团队推荐总量、团队转化率、客户公司数、项目数），展示团队核心指标的实时汇总数据，每个卡片显示对比趋势。中部为日期面板/周历看板切换区域，支持按日/周视图切换，展示今日组员获取、今日推荐简历、今日客户处理岗位、今日客户数、今日项目数等团队指标，以及客户公司数、岗位数、求职者数量、总推荐数、总操作员数等通用统计项，每项均可点击下钻查看组员明细。下方为快捷入口区域，包含8个常用功能模块的图标按钮（求职者数据池、简历导入、客户管理、项目管理、评价管理、标签字典、质保期管理、统计管理）。再下方为左右两列布局，左侧为下属操作员绩效排行（Top 5），右侧为负责客户公司列表（4家）。最下方为项目岗位进度（4项，带进度条）和组长待办事项列表（4项）。页面整体采用卡片式布局，支持点击下钻形成三级数据穿透链路（汇总->明细表->条件筛选列）。页面展示的数据范围严格受组长数据权限控制，仅显示授权范围内的团队数据。

###### 4.3 功能项详细描述

首页-数据看板（组长）功能项包含以下10个功能点，覆盖组长工作台的核心交互：

1. 统计卡片区域——展示6项团队指标（简历总量、今日新增、团队推荐总量、团队转化率、客户公司数、项目数）实时汇总数据，支持点击下钻到明细数据，卡片显示对比趋势。

2. 日期面板视图——按日查看团队运营指标，支持左右箭头翻页和快速导航到当天或本周，面板显示今日组员获取、今日推荐简历、今日客户处理岗位、今日客户数、今日项目数等团队指标，以及客户公司数、岗位数、求职者数量、总推荐数、总操作员数等通用统计项。

3. 周历看板视图——按周对比团队指标趋势，显示本周7天的每日推荐、面试等数据汇总，以及周汇总卡片（本周客户公司数、本周岗位数、本周求职者数量、本周总推荐数、本周总操作员数），支持日期切换。

4. 快捷入口——8个常用功能模块的快捷访问按钮，包括求职者数据池、简历导入、客户管理、项目管理、评价管理、标签字典、质保期管理、统计管理，点击跳转对应功能模块。

5. 下属操作员绩效排行——展示组员的工作效率和推荐量排名，按团队推荐量、转化率等指标综合排序，展示Top 5操作员数据，帮助组长掌握团队成员表现。

6. 负责客户公司——展示团队负责的客户公司项目，包括客户公司名称、合作状态、项目数量、当前处理进度等信息，帮助组长了解项目整体进展。

7. 项目岗位进度——以进度条形式展示各项目的岗位招聘完成状态，包括项目名称、岗位名称、当前进度、完成数量和总数量，帮助组长跟踪项目进展。

8. 组长待办事项——展示需要组长处理的待办任务列表，包括待审核的简历推荐、新匹配简历待审核、低于平均的组员需跟进、紧急岗位处理等4项待办任务，高优先级置顶显示。

9. 日期面板汇总下钻——点击日期面板中的任意汇总卡片，下钻查看对应组员明细数据，如点击"今日组员获取"下钻到组员明细表，点击"今日推荐简历"下钻到推荐明细表。

10. 三级下钻筛选——明细表->条件筛选列的多级数据穿透，点击明细表中的任意一行可下钻至条件筛选列，支持多条件组合筛选（AND关系），筛选结果以标签形式展示已选条件，支持清空和重置筛选。

#### 首页-数据看板（操作员）

##### 1、业务需求

作为一线招聘操作员，日常需要处理简历导入、候选人推荐、面试安排等工作。首页数据看板为操作员展示个人工作数据的实时概览，帮助操作员了解今日的工作进展和待处理事项。通过统计卡片展示个人负责简历总量、今日新增、今日面试、个人推荐量、个人转化率、待处理任务等6项个人指标，操作员可以快速掌握自己的工作节奏。日期面板展示今日面试、今日入职、质保结束等个人级指标。快捷入口提供常用功能模块的快速访问，待办事项提醒需要优先处理的任务。

##### 2、功能说明

首页-数据看板（操作员）是面向操作员角色的个人工作台页面。该功能模块为操作员提供个人工作数据的集中展示：（1）顶部统计卡片展示简历总量、今日新增、今日面试、个人推荐量、个人转化率、待处理任务等6项个人指标，并显示对比趋势；（2）日期面板区域支持按日/周切换，展示今日面试、今日入职、今日质保结束等个人指标，每项均可下钻查看明细；（3）快捷入口提供求职者数据池、简历导入、项目管理、评价管理等4个常用功能模块入口；（4）个人工作台下方展示操作员负责的岗位列表和进度、个人通知列表和个人待办事项。页面展示的数据范围严格受操作员数据权限控制，仅显示个人负责的数据。操作员可以通过首页快速了解今日工作量和任务完成情况，快捷处理待办事项。
