# 模块二：求职者数据池

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：统一管理候选人简历资源，支持搜索、筛选、预览、状态、评价、标签、导出和跟踪入口。

**本模块负责**：

- 简历搜索与多条件筛选
- 候选人卡片、预览、标签、评价
- 锁定/释放、激活/失效、导出
- 候选人跟踪表入口

**本模块不负责**：

- 客户公司和项目岗位主数据维护
- 最终推荐交付记录管理
- 权限规则配置

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

求职者数据池是招聘管理系统的核心模块，负责统一管理所有候选人的简历数据。该模块提供简历的存储、检索、筛选、跟踪、评价和状态管理的完整功能，支持AI简历解析和自动匹配能力。

## 功能拆解

### 1. 求职者全局搜索

- 功能描述：高效管理和运营候选人资源。进入求职者数据池 → 在搜索框输入关键词 → 支持职位+关键词联合搜索 → 查看「猜你想搜」智能推荐 → 执行搜索
- 菜单/入口：求职者数据池 → 求职者列表 → 全局搜索
- 界面元素：搜索关键词（公司、职位、专业、学校、行业、技能等）
- 输出结果：匹配的简历列表
- 权限说明：所有用户按数据权限范围搜索

**操作流程**

1. 进入求职者数据池
2. 在搜索框输入关键词
3. 支持职位+关键词联合搜索
4. 查看「猜你想搜」智能推荐
5. 执行搜索

**输入字段**

<!-- table:1 rows:3 cols:4 -->
| 字段 | 类型 | 必填 | 校验规则 |
| --- | --- | --- | --- |
| 期望城市 | 文字 | - | 根据行政区划输入全称或拼音或首字母显示城市名称（名称包含省市） |
| 公司、职位、专业、学校、行业等条件 | 文字 | - | 多种条件组合填写，根据填写的字段按顿号分割查询 |

**业务规则**

- 支持全文检索；支持【职位+关键词】联合搜索；支持高级搜索入口；支持猜你想搜智能推荐；搜索结果按相关性排序

**验收关注点**

- 入口、按钮、表单字段和权限表现与产品规格一致。
- 成功态、空态、失败态、权限不足态均有明确表现。
- 关键操作写入操作日志；涉及数据列表时必须按角色数据范围过滤。
- 与其他模块联动时，状态变化和输出数据可被后续模块消费。

### 3. 简历单文件导入

- 功能描述：高效管理和运营候选人资源。进入简历导入页面 → 选择单文件上传（Word/PDF）→ 系统自动解析 → 提取结构化信息 → 执行去重对比（姓名+身份证号 或 姓名+手机号）→ 若存在相同简历弹出对比页面 → 操作者判断是否覆盖 → 导入成功

- 输出结果：结构化存储的简历数据、原始文件附件

**操作流程**

1. 进入简历导入页面
2. 选择单文件上传（Word/PDF）
3. 系统自动解析
4. 提取结构化信息
5. 执行去重对比（姓名+身份证号 或 姓名+手机号）
6. 若存在相同简历弹出对比页面
7. 操作者判断是否覆盖
8. 导入成功

- 支持Word和PDF格式；自动提取姓名、身份证号、手机号、邮箱、年龄、性别、学历等18+结构化字段；去重依据：姓名+身份证号 或 姓名+手机号 或 姓名+手机号+教育经历；相同简历弹出对比页面，操作者可判断是否覆盖；导入后简历状态为「激活」；导入文件带水印

求职者全局搜索是求职者数据池是招聘管理系统的核心模块，负责统一管理所有候选人的简历数据。该模块提供简历的存储、检索、筛选、跟踪、评价和状态管理的完整功能，支持AI简历解析和自动匹配能力。中的重要功能组件。该功能主要解决以下业务问题：

在人力公司招聘管理业务场景中，求职者数据管理需要此功能来高效运营候选人资源。通过求职者全局搜索功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

在人力公司招聘管理业务场景中，求职者数据管理需要此功能来高效运营候选人资源。通过简历导出功能，系统实现了业务效率的提升和数据管理的规范化，显著提升了业务操作效率和数据管理的规范性。

简历导出功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如标签字典管理、项目管理、候选人跟踪表）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

简历导出功能定位于求职者数据管理的核心功能，是简历导出体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持简历的高效导入、结构化存储和快速检索
2. 支持多维度条件筛选、状态管理和评价功能
3. 操作结果实时生效，数据自动更新

简历导出功能与其他功能的关联关系：该功能与与标签字典、项目管理、候选人跟踪表联动提供完整的候选人管理紧密关联，共同支撑候选人数据的全生命周期管理流程的完整业务流程。通过简历导出功能，操作员可以高效管理和运营候选人资源，为后续的候选人推荐、面试和交付工作奠定数据基础。

功能权限方面，该功能支持不同角色的差异化数据访问，管理员全量、组长授权范围、操作员个人范围。所有操作均记录操作日志，确保操作的可追溯性。
