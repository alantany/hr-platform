# 模块十二：标签字典管理

> 本分册是可独立交给开发 agent 的模块 PRD。权威来源为 `招聘管理系统_功能详细清单_v3.0.docx`；若与旧原型理解或现有页面实现冲突，以产品最终 DOCX 为准。

## Agent 任务包

**任务目标**：统一维护求职者、客户需求和评价体系相关标签字典，支撑筛选、岗位要求、评价和统计。

**本模块负责**：

- 求职者标签字典
- 客户需求标签字典
- 评价体系标签字典
- 标签搜索与导出

**本模块不负责**：

- 使用标签的业务动作本身
- 权限角色配置
- AI 匹配算法

**必须先读取的横切文档**：

- `PRD.md`
- `docs/prd/90-data-fields.md`
- `docs/prd/91-permission-matrix.md`

**建议并行开发拆分**：

- 前端 agent：页面、表单、列表、筛选、状态展示、空态/失败态。
- 后端 agent：API、权限过滤、状态流转、操作日志、导入/导出或异步任务。
- 数据库 agent：模型、索引、枚举、关联表、迁移和种子数据。
- 测试 agent：接口级验收、权限矩阵、状态流转、边界条件和失败场景。

## 模块定位

标签字典管理模块是系统的基础配置模块，由管理员专属使用。该模块统一管理系统中所有标签字典数据，包括求职者标签、客户需求标签和评价体系标签，为其他模块提供标准化的标签数据源。

### 上游依赖

- 权限管理体系
- 求职者数据池
- 项目管理

- 对通知、统计、AI 或导出模块输出明确的业务事件或结果数据。

## 功能拆解

### 1. 求职者标签字典-管理

- 功能描述：维护系统标签字典的标准数据。进入标签字典管理 → 选择「求职者标签」分类 → 查看/新增/编辑/删除标签 → 保存
- 菜单/入口：系统设置 → 标签字典管理 → 求职者标签字典
- 界面元素：标签分类、标签值、排序、状态（启用/禁用）
- 输出结果：更新后的求职者标签体系
- 权限说明：仅超级管理员可管理标签字典

1. 进入标签字典管理
2. 选择「求职者标签」分类
3. 查看/新增/编辑/删除标签
4. 保存

## 推荐接口与后端边界

- 列表查询接口必须支持分页、排序、筛选、关键词搜索和数据权限过滤。
- 详情接口必须校验当前用户是否可访问该业务对象。
- 新增/编辑/删除/状态流转接口必须记录操作日志。
- 批量、导入、导出、AI、通知等长任务建议建任务表或事件表，保留状态、错误信息和重试线索。
- 所有枚举值优先从产品规格和标签字典中抽取，不允许前后端各自硬编码不同口径。

## 数据与权限要求

- 字段定义以 `docs/prd/90-data-fields.md` 为准；模块内表格字段是功能输入/输出的补充。
- 功能权限、数据权限、菜单可见性以 `docs/prd/91-permission-matrix.md` 为准。
- 菜单隐藏不能替代后端权限校验；所有 API 都必须执行权限判断。
- 导出、统计、AI 推荐结果、通知跳转必须与列表/详情使用同一套数据权限过滤。

##### 1、业务需求

求职者标签字典-管理是标签字典管理模块是系统的基础配置模块，由管理员专属使用。该模块统一管理系统中所有标签字典数据，包括求职者标签、客户需求标签和评价体系标签，为其他模块提供标准化的标签数据源。中的重要功能组件。该功能主要解决以下业务问题：

在人力公司招聘管理业务场景中，管理员需要维护求职者标签字典体系，包括年龄规则、学历规则、工作经验等维度，为系统筛选和匹配提供数据基础。通过求职者标签字典功能，系统实现了求职者标签的标准化和统一化管理，显著提升了业务操作效率和数据管理的规范性。

求职者标签字典功能的设计充分考虑了不同角色用户（超级管理员、组长、操作员）的实际操作需求，确保功能定位准确、业务逻辑清晰、操作便捷高效。同时，该功能与其他模块（如求职者数据池、评价体系、项目管理）形成紧密的业务关联，共同构成完整的招聘管理闭环。

##### 2、功能说明

求职者标签字典-管理功能定位于系统基础配置模块的核心功能，是求职者标签字典体系中的核心功能组件。

该功能提供以下核心能力：
1. 支持标签的新增、编辑、删除和分类管理
2. 支持标签的搜索、导出和批量操作
3. 标签变更实时同步到所有关联模块

求职者标签字典功能与其他功能的关联关系：该功能与与求职者数据池、项目管理、评价体系联动提供标签数据源紧密关联，共同支撑标签字典的维护和管理流程的完整业务流程。通过求职者标签字典功能，操作员可以维护系统标签字典的标准数据，为后续的候选人筛选、岗位匹配和简历评价奠定数据基础。

功能权限方面，标签字典的增删改操作仅超级管理员可用，组长和操作员仅有只读权限。所有操作均记录操作日志，确保操作的可追溯性。
